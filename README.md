## About this repo
This repository is prepared as a job-application portfolio to showcase real-world manual QA artifacts:
- Structured test cases covering core E-Expense flows
- Traceability to requirements
- Regression checklist for release readiness
- Professional bug reporting template for reproducible issues

## How to use
1. Read `/docs/TestPlan.md` to understand scope and approach.  
2. Use templates in `/templates/` when creating or executing test cases.  
3. Execute test cases in SIT / UAT environment and attach evidence under `evidence/screenshots` (optional).  
4. Log defects using GitHub Issues — use the provided issue template for consistency.  
5. Keep the traceability matrix updated when requirements change.
